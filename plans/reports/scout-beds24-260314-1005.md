## Architecture

### Flow Diagram
```
Client (OpenClaw/LLM)
  ↓
HTTP Gateway (port 8767)
  ↓ (MCP Streamable HTTP)
MCP Server (port 8001)
  ↓ (Beds24 API)
Beds24 API v2
```

### Token Management
- **Long Life Token**: Read-only, không expire nếu dùng trong 90 ngày
- **Refresh Token**: Read/write, expire sau 30 ngày inactivity
- **Auto-refresh**: Access token được tự động refresh từ refresh token
- **Session Management**: Gateway maintain mcp-session-id cho connections

## Environment Variables

### Required
- `BEDS24_API_TOKEN` hoặc `BEDS24_REFRESH_TOKEN` - Authentication token

### Optional
- `BEDS24_API_BASE_URL` - API base URL (default: https://api.beds24.com/v2)
- `GATEWAY_PORT` - Gateway port (default: 8767)
- `BEDS24_MCP_URL` - MCP server URL (default: http://localhost:8001)

## Setup Flow

1. **Generate Invite Code**: https://beds24.com/control3.php?pagetype=apiv2
2. **Call Setup Tool**: `beds24_setup_from_invite_code` với invite code
3. **Save Refresh Token**: Lưu refresh_token vào environment variable
4. **Start Services**:
   ```bash
   cd openclaw-skills/beds24-mcp
   docker compose up --build -d
   ```

## Security Notes

- **Never commit tokens**: BEDS24_API_TOKEN, BEDS24_REFRESH_TOKEN không được commit
- **Rate limiting**: Default 100 credits/5 minutes
- **Session timeout**: 10s healthcheck với 3 retries
- **Auto-reinit**: Session tự động reinitialize khi expired
